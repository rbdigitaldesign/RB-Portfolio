**Miller's Law** — named after psychologist **George Miller** — states that the average person can hold approximately **7 (plus or minus 2) items** in their working memory at any given time.

It comes from Miller's 1956 paper *"The Magical Number Seven, Plus or Minus Two: Some Limits on Our Capacity for Processing Information"* — one of the most cited papers in the history of psychology. Miller described working memory as having a limited number of "slots," and introduced the concept of **chunking** as the primary strategy for working within that limit.

**Chunking:**
A "chunk" is a meaningful unit of information. The key insight is that a chunk can contain multiple pieces of information: the string "R-O-Y-G-B-I-V" is seven letters (seven chunks), but once you learn that it stands for the colours of the spectrum, it becomes a single chunk ("rainbow order"). Chunking allows more information to be held in working memory by repackaging it into larger, meaningful units.

This is why phone numbers are formatted in groups rather than as continuous strings, why passwords are easier to remember when they include recognisable words, and why navigation taxonomies use categories rather than flat lists.

**The updated number:**
More recent research, notably by Nelson Cowan (2001), suggests the true capacity of working memory is closer to **4 chunks** rather than 7. Miller's number has been revised downward — but the core implication (working memory is severely limited and design must respect this) remains unchanged.

**Design implications:**

*Navigation:* Primary navigation menus should contain 5-7 items. Below 5, users may feel options are missing. Above 7, the cognitive scan cost increases. Sub-menus allow complexity to be chunked into manageable groups.

*Forms:* Long forms should be broken into sections or wizard steps, with related fields grouped. Presenting 25 fields on a single page exhausts working memory; presenting them in 4 grouped steps of 5-6 fields each is manageable.

*Pricing:* Pricing pages with 3-4 tiers are more decision-friendly than those with 8. Choice evaluation is a working memory task — more choices require more simultaneous comparisons.

*Onboarding:* Introducing 3 new concepts at a time is manageable; introducing 10 simultaneously is not. Sequencing matters: new information can't be chunked until it's understood, so high working memory loads at the start of an experience are particularly costly.

**In learning design:**
Working memory limits are foundational to cognitive load theory (Sweller, 1988). Miller's Law underlies the principle that learning outcomes should be stated in sets of 3-5 per module, that content should be chunked into clearly bounded topics, and that prior knowledge is valuable because it allows new information to be assimilated into existing chunks rather than held as isolated items.

A student encountering dense new content without scaffolding is holding too many unconnected items in working memory simultaneously — the classic extraneous load problem. Chunking, sequencing, and worked examples are all strategies for working within Miller's limit.
