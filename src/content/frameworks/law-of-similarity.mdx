The **Law of Similarity** is a Gestalt principle stating that elements that share visual characteristics — shape, colour, size, texture, orientation — are perceived as belonging to the same group, regardless of their spatial position.

It was articulated by **Max Wertheimer** in his 1923 paper on perceptual grouping, and it underlies much of what is now formalised in design systems thinking.

**Why it works:**
The brain uses visual resemblance as a proxy for categorical membership. If multiple elements share the same appearance, the inference is that they are *of a kind* — they belong to the same group, serve the same function, or represent the same type of entity. This is an efficient heuristic: in the natural and designed world, things that look alike often do work alike.

**The implications for interface design:**

*Affordances through consistency:* If all interactive elements in an interface share a consistent visual style, users don't need to consciously evaluate each one. "I've learned that blue underlined text is a link — every instance of blue underlined text is therefore a link." Violating this (having some blue underlined text that *isn't* a link) breaks the learned model and creates confusion.

*Design systems:* The Law of Similarity is the perceptual psychology behind design systems. A design system isn't just about efficiency of production — it's about cognitive efficiency for users. Consistent components mean users encounter familiar patterns, reducing the overhead of evaluation and re-learning.

*Colour coding:* When a colour consistently means the same thing (red = error, green = success, amber = warning), users learn the code and apply it automatically. Inconsistent application of colour — red errors in some contexts, red headings in others — destroys the signalling value of the colour.

*Visual hierarchy:* Headings at the same level look alike. Body text looks consistent. Captions look consistent. This visual grammar allows users to navigate and categorise content rapidly.

**Similarity vs. proximity:**
Similarity can override proximity. If elements share the same colour but are spatially separated, they will still be perceived as grouped — the eye will "connect" them across the space between them. This is useful for creating coherent visual groupings across complex layouts.

**Violation as a tool:**
Breaking similarity deliberately signals difference. When an element looks unlike all the others, it communicates: "this is in a different category." This is how warning messages, calls-to-action, and status indicators use visual distinction to draw attention. The Von Restorff Effect is essentially the perceptual consequence of breaking the Law of Similarity in a controlled way.

**In learning design:**
Consistent visual treatment for content types — all key terms styled the same way, all activities in the same card format, all warnings in the same callout style — creates a visual language students can learn and apply. Once learned, the style communicates content type before the student reads a word.
